Local OCR API

A self-contained, local-only OCR (Optical Character Recognition) service designed for screen reading and AI agents. It runs Tesseract OCR inside Docker and exposes a high-performance HTTP endpoint using FastAPI.

🔒 100% Local

No internet required, no third-party APIs, and no cloud latency. Your data never leaves your machine.

⚡ Performance

Extremely fast OCR transcription on standard screenshots.
Average Latency: ~300ms

📷 Screenshot Preprocessing

Performs automatic grayscale, autocontrast, and smart upscaling to yield ultra-accurate readings on small UI labels.

🤖 Built for Agents

Features endpoints designed for file uploads and base64-encoded strings (perfect for in-memory LLM pipelines).

API Endpoints

  • POST /ocr — Send image file as multipart/form-data.
  • POST /ocr/base64 — Send image as base64 string JSON.
  • GET /health — Service status and Tesseract version details.

Agent Skill System Prompt

Copy the following system instruction block to define this capability for your local AI agent:

# System Skill: Read User Screenshot
You are equipped with a local OCR tool to view the user's screen.
When you need to read a screenshot or image file, invoke the local OCR API by sending a HTTP POST request.

- Endpoint: http://localhost:8000/ocr
- Method: POST
- Body Format: Multipart Form-Data
- Parameter Name: "file" (The binary image file)

Alternatively, if you hold the image in memory as a base64 encoded string, use:
- Endpoint: http://localhost:8000/ocr/base64
- Method: POST
- Body Format: JSON
- Payload Structure: {"image_base64": "YOUR_BASE64_STRING"}

Quick Start

docker compose up -d --build
curl -s localhost:8000/health
